# AppEngine Charter

## Purpose

AppEngine is an autonomous venture-building command center. It turns ideas, GitHub issues, monitoring signals, customer needs, and agent findings into structured plans, prompts, pull requests, QA loops, deployment readiness, and follow-up work.

## Primary User

Lincoln, acting as visionary, builder, operator, and steward of multiple apps and ventures.

## What AppEngine Should Do

- Preserve source-of-truth context across ChatGPT, Codex, GitHub, and future agents.
- Plan apps and features from rough ideas.
- Generate Codex-ready prompts from manifest-backed agent context.
- Route work through agent workflows.
- Create reviewable pull requests.
- Record QA, deployment readiness, blockers, and follow-up issues.
- Connect opportunities across the broader app ecosystem when useful.

## What AppEngine Should Not Become

- A system that bypasses human approval for production-impacting changes.
- A place where private chat memory becomes the only source of truth.
- A generic app factory that ignores app-specific charters and boundaries.
- A deployment tool that exposes secrets or grants broad production permissions to AI agents.

## Success Definition

AppEngine succeeds when Lincoln can start with a messy idea in ChatGPT, move durable context into GitHub, trigger the right agent workflow, review a pull request, and keep future work aligned with the original mission and app boundaries.
